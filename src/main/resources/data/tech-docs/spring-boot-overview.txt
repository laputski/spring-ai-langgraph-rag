Spring Boot Overview
====================

Spring Boot is an opinionated framework built on top of the Spring Framework that simplifies the creation of stand-alone, production-grade Spring-based applications. It takes an "opinionated defaults" approach to configuration, reducing the amount of boilerplate code developers need to write.

Key Features
------------

Auto-Configuration
Spring Boot automatically configures Spring application based on the dependencies present in the classpath. For example, if spring-data-jpa is on the classpath and a DataSource bean is configured, Spring Boot will automatically configure a JPA repository.

Embedded Server
Spring Boot includes embedded web servers (Tomcat, Jetty, Undertow), eliminating the need to deploy WAR files. Applications are packaged as executable JARs and can be run with `java -jar`.

Starter Dependencies
Spring Boot Starters are a set of convenient dependency descriptors that bundle commonly used dependencies together. For example, `spring-boot-starter-web` includes Spring MVC, embedded Tomcat, and Jackson.

Spring Boot Actuator
Actuator adds production-ready features such as health checks, metrics, auditing, and HTTP tracing. Endpoints are exposed over HTTP or JMX.

Spring Boot DevTools
DevTools provides fast application restarts, live reload, and remote development configurations for an improved developer experience.

Auto-Configuration Example
---------------------------

When you add `spring-boot-starter-data-redis` to your project, Spring Boot auto-configures:
- LettuceConnectionFactory (default Redis client)
- RedisTemplate<Object, Object>
- StringRedisTemplate

You only need to provide connection properties in application.properties or application.yml:

  spring.data.redis.host=localhost
  spring.data.redis.port=6379

Spring Boot 3.x Changes
------------------------

Spring Boot 3.x requires Java 17 or later and is based on Spring Framework 6.x. Key changes include:
- Migration from javax.* to jakarta.* namespace (Jakarta EE 9+)
- Native compilation support via GraalVM
- Observability improvements with Micrometer
- Improved AOT (Ahead of Time) processing
- ProblemDetail support for RFC 7807 error responses
